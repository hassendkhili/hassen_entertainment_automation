# Maestro Automation - Core Logic

## 1. Time Monitoring (المراقب الزمني)
يتم استخدام `Timer.periodic` ليعمل كل ثانية واحدة.
- يقوم بتحديث واجهة الساعة (UI).
- يستدعي دالة `_checkAndPlayAudio()` للتحقق من المواعيد.

## 2. Execution Logic (منطق التشغيل)
في كل ثانية، يتم مقارنة `الوقت الحالي` مع `وقت النشاط` في القائمة.
- **إذا تطابق الوقت (Match):** يتم التحقق من متغير `hasPlayed`.
- **إذا كان `hasPlayed == false`:** يتم تشغيل الصوت، وتغيير المتغير إلى `true` لمنع تكرار التشغيل في نفس الدقيقة.
- **عند تغير الدقيقة:** يعود المتغير `hasPlayed` إلى `false` ليكون جاهزاً لليوم التالي.

## 3. Data Persistence (حفظ البيانات)
- نستخدم مكتبة `shared_preferences`.
- يتم تحويل كائن `ActivitySchedule` إلى `Map` ثم إلى `JSON String` للحفظ.
- عند فتح التطبيق، يتم استرجاع الـ `JSON` وتحويله مجدداً إلى كائنات لعرضها.
